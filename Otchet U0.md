# Отчёт по заданию U0: Настройка окружения

## Цели задания
Настроить полное окружение:
1. Браузер для работы и поиска информации
2. Интегрированную среду разработки (IDE)
3. Java Development Kit (JDK) и среду выполнения (JRE)
4. Систему сборки проектов 
5. Драйвер для автоматизации браузера
6. Систему контроля версий

## Часть 1: Установка Google Chrome

### Процесс установки

1. **Переход на официальный сайт**  
   Открыть любой браузер и перейти на страницу загрузки Google Chrome: `https://www.google.com/chrome`.
   <br>![Страница загрузки Chrome](screenshots/chrome-download-page.png)
   <br>*Скриншот 1: «Страница загрузки Chrome» — виден интерфейс сайта с кнопкой «Скачать Chrome»*

2. **Скачивание и запуск установщика**  
   Нажать на кнопку «Скачать Chrome». Начинается загрузка файла установщика `ChromeSetup.exe`. После завершения загрузки запустить его двойным щелчком мыши.
   <br>![Процесс скачивания установщика](screenshots/download-progress.png)
   <br>*Скриншот 2: «Процесс скачивания» — окно загрузки в браузере с именем файла и индикатором прогресса*

3. **Процесс установки**  
   Установщик автоматически начинает распаковку и установку браузера. Никаких дополнительных настроек выбирать не требуется. После окончания процесса установщик закрывает своё окно, а на рабочем столе появляется ярлык Google Chrome.
   <br>![Процесс установки Chrome](screenshots/installation-progress.png)
   <br>*Скриншот 3: «Ход установки» — окно установщика с индикатором прогресса*

4. **Первый запуск браузера**  
   Двойным щелчком открыть ярлык Chrome. Появляется стартовое окно браузера.
   <br>![Первое открытие Chrome](screenshots/first-launch.png)
   <br>*Скриншот 4: «Первое открытие Chrome» — открытое окно браузера с начальной страницей*

5. **Проверка версии браузера**
    - Нажать на три точки (меню) в правом верхнем углу.
    - Выбрать «Справка» → «О браузере Google Chrome».  
      На открывшейся странице отображается текущая версия.
      <br>![Версия браузера Chrome](screenshots/browser-version-info.png)
      <br>*Скриншот 5: «Страница „О браузере“» — экран с указанием версии Chrome*

## Часть 3: Установка Java JDK и JRE, настройка переменных окружения

## Цели части

- Установить Java Development Kit (JDK) и Java Runtime Environment (JRE).
- Настроить системные переменные окружения.
- Проверить корректность установки и настройки.

## Процесс установки JDK и JRE

### 1. Загрузка установщика

- Перейти на официальный сайт Oracle для загрузки JDK.
- Выбрать подходящую версию для операционной системы.
- Загрузить установочный файл `jdk-8u471-windows-x64.exe`.

  <br>![Скриншот 15: «Страница загрузки JDK»](screenshots/jdk-download-page.png)  
  <br>*Скриншот 15: «Страница загрузки JDK» — виден интерфейс сайта с кнопкой загрузки.*

### 2. Начало установки JDK 8

- Запустить скачанный установочный файл `jdk-8u471-windows-x64.exe`.
- Внимательно ознакомиться с важными изменениями:
    - Лицензионное соглашение обновлено.
    - Java Mission Control (JMC) больше не включён в состав JDK.
    - JavaFX больше не включён в состав JDK.
- Нажать кнопку **Next**.

<br>![Скриншот 16: «Окно мастера установки JDK 8»](screenshots/jdk-setup-wizard.png)  
<br>*Скриншот 16: «Окно мастера установки JDK 8» — начальный экран с важной информацией об изменениях.*

### 3. Выбор компонентов для установки

- Оставить все компоненты по умолчанию:
    - Development Tools (основные инструменты разработки);
    - Source Code (исходный код);
    - Public JRE (общедоступная среда выполнения).
- Нажать кнопку **Next**.
- Ожидать завершения процесса установки.
> **Важно:** Запомнить путь установки, выделенный в окне (в данном случае — `C:\Program Files\Java\jdk-1.8\`). 

  <br>![Скриншот 17: «Выбор компонентов и путь установки»](screenshots/jdk-custom-setup.png)  
  <br>*Скриншот 17: «Выбор компонентов и путь установки» — виден путь `C:\Program Files\Java\jdk-1.8\`.*

  <br>![Скриншот 18: «Установка Java»](screenshots/jdk-installation-progress.png)  
  <br>*Скриншот 18: «Установка Java» — индикатор выполнения установки.*

### 4. Выбор папки для JRE (если выбрана)

> **Примечание:** Этот шаг появляется, если на шаге 2 был выбран компонент *Public JRE*.

- Появляется окно выбора папки для установки JRE.
- По умолчанию предлагается путь: `C:\Program Files\Java\jre1.8.0_471`.
- При необходимости изменить путь:
    - нажать кнопку **Change**;
    - выбрать нужную директорию;
    - подтвердить выбор.
- Нажать кнопку **Next** для продолжения установки.

![Скриншот 19: «Выбор папки для JRE»](screenshots/jdk-jre-destination.png)  
*Скриншот 19: «Выбор папки для JRE» — указание пути установки среды выполнения Java.*


### 5. Активная установка Java

- Непосредственная установка Java в систему.
- Отображается индикатор с текстом **Status: Installing Java**.
- Идёт копирование файлов и их регистрация в системе.

![Скриншот 20: «Установка Java»](screenshots/jdk-installing-java.png)  
*Скриншот 20: «Установка Java» — активная фаза установки с логотипом Oracle.*
### 6. Завершение установки

- После успешной установки появится соответствующее уведомление.
- Появится предложение перейти к документации и руководствам.
- Нажать кнопку **Close** для завершения установки.

  <br>![Скриншот 21: «Завершение установки JDK»](screenshots/jdk-installation-complete.png)  
  <br>*Скриншот 21: «Завершение установки JDK» — окно с сообщением об успешной установке.*

## Настройка переменных окружения

### 1. Открыть окно системных переменных

- Перейти в **Панель управления** → **Система** → **О системе**.
- Выбрать **Дополнительные параметры системы**.
- Нажать кнопку **Переменные среды**.

### 2. Создание переменной `JAVA_HOME`

- Нажать кнопку **Создать** в разделе системных переменных.
- Ввести **имя переменной**: `JAVA_HOME`.
- Ввести **значение переменной**: `C:\Program Files\Java\jdk-1.8` (путь, который нужно было запомнить на шаге 3).

  <br>![Скриншот 22: «Создание переменной JAVA_HOME»](screenshots/create-java-home.png)  
  <br>*Скриншот 22: «Создание переменной JAVA_HOME» — окно ввода имени и значения.*

### 3. Добавление пути в переменную `PATH`

- В списке системных переменных выбрать переменную **`Path`**.
- Нажать кнопку **Изменить**.
- Добавить новую запись: `%JAVA_HOME%\bin`.
- Подтвердить изменения нажатием **ОК** во всех открытых окнах.

  <br>![Скриншот 23: «Изменение переменной PATH»](screenshots/edit-path-variable.png)  
  <br>*Скриншот 23: «Изменение переменной PATH» — добавление `%JAVA_HOME%\bin`.*

### 4. Создание переменной `JAVA`

- Нажать кнопку **Создать** в разделе системных переменных.
- Ввести **имя переменной**: `JAVA`.
- Ввести **значение переменной**: `%JAVA_HOME%\bin`.

  <br>![Скриншот 24: «Создание переменной JAVA»](screenshots/create-java-variable.png)  
  <br>*Скриншот 24: «Создание переменной JAVA» — окно ввода имени и значения.*

## Проверка установки

- Запуск `cmd.exe`.
- Выполнение следующих команд и анализ результатов:
```bash
java -version
javac -version
```

<br>![Скриншот 25: «Проверка версий Java»](screenshots/java-version-check.png)  
<br>*Скриншот 25: «Проверка версий Java» — вывод команд `java -version` и `javac -version`.*

## Часть 4: Установка и настройка Apache Maven

## Цели части
- Установить систему сборки проектов Apache Maven.
- Настроить переменные окружения для работы Maven из командной строки.
- Проверить корректность установки и настройки.

### 1. Скачивание дистрибутива Apache Maven

- Перейти на официальный сайт Apache Maven: [https://maven.apache.org/download.cgi](https://maven.apache.org/download.cgi).
- Скачать архивную версию в формате ZIP под названием `apache-maven-v.v.v-bin.zip`.

<br>![Скриншот 26: «Загрузка архива Apache Maven с официального сайта»](screenshots/maven-download-page.png)  
<br>*Скриншот 26: «Загрузка архива Apache Maven с официального сайта» — интерфейс сайта с выделенной ссылкой для скачивания бинарного дистрибутива.*

### 2. Распаковка архива Maven

- Создать директорию: `C:\Program Files\Maven`.
- Распаковать содержимое скачанного архива в созданную директорию.
- В результате в указанной папке должна появиться структура с подкаталогами:
    - `bin`
    - `conf`
    - `lib`
    - и другими.

<br>![Скриншот 27: «Содержимое распакованной директории Maven»](screenshots/maven-unpacked-folder.png)  
<br>*Скриншот 27: «Содержимое распакованной директории Maven» — проводник Windows с отображением распакованных файлов Maven в `C:\Program Files\Maven\apache-maven-3.9.12`.*

### 3. Создание системной переменной MAVEN_HOME

- Открыть окно **Переменные среды**:  
  Перейти в **Панель управления** → **Система** → **О системе**.
- Выбрать **Дополнительные параметры системы**.
- Нажать кнопку **Переменные среды**.
- В разделе **Системные переменные** нажать кнопку **Создать**.
- Ввести **имя переменной**: `MAVEN_HOME`.
- Ввести **значение переменной**: `C:\Program Files\Maven\apache-maven-3.9.8` (укажите актуальный путь к распакованной папке Maven).

<br>![screenshots/maven-home-variable.png](screenshots/maven-home-variable.png)  
<br>*Скриншот 28: «Создание переменной окружения MAVEN_HOME» — окно настройки переменной с указанием пути к установленному Maven.*

### 4. Добавление Maven в системный путь PATH

- В списке системных переменных выбрать переменную **`Path`**.
- Нажать кнопку **Изменить**.
- Добавить новую запись: `%MAVEN_HOME%\bin`.
- Подтвердить изменения нажатием **ОК** во всех открытых окнах.

<br>![Скриншот 29: «Добавление пути Maven в переменную PATH»](screenshots/maven-path-variable.png)  
<br>*Скриншот 29: «Добавление пути Maven в переменную PATH» — окно редактирования переменной PATH с новой записью `%MAVEN_HOME%\bin`.*

### 5. Проверка работоспособности Maven

- Запустить новое окно командной строки (`cmd.exe`).
- Выполнить команду:
```bash
mvn -version
```
<br>![Скриншот 30: «Проверка установки Maven выполнением команды mvn -version»](screenshots/maven-version-check.png)

## Часть 5: Установка ChromeDriver для автоматизации браузера

## Цели части

- Скачать и настроить ChromeDriver для автоматизации тестов в Google Chrome.
- Добавить путь к драйверу в системные переменные окружения.
- Проверить корректность установки драйвера.

## Процесс установки ChromeDriver

### 1. Скачивание ChromeDriver с официального сайта

- Перейти на официальную страницу загрузки ChromeDriver: [https://chromedriver.chromium.org/downloads](https://chromedriver.chromium.org/downloads).
- Выбрать версию драйвера, совместимую с установленной версией Google Chrome.
- Скачать архив для Windows (`chromedriver_win64.zip`).

<br>![Скриншот 31: «Страница загрузки ChromeDriver с официального сайта»](screenshots/chromedriver-download-page.png)  
<br>*Скриншот 31: «Страница загрузки ChromeDriver с официального сайта» — интерфейс сайта с доступными версиями драйвера и ссылками для скачивания.*

### 2. Создание системной папки для веб‑драйверов, извлечение и размещение драйвера

- Открыть проводник Windows и перейти в корень диска `C:\`.
- Создать новую папку с именем `webdrivers`.
- Распаковать скачанный архив `chromedriver_win64.zip`.
- Переместить файл `chromedriver.exe` в созданную папку: `C:\webdrivers`.

<br>![Скриншот 32: «Файл chromedriver.exe в папке C:\webdrivers»](screenshots/create-webdrivers-folder.png)  
<br>*Скриншот 32: «Файл chromedriver.exe в папке C:\webdrivers» — содержимое папки с размещённым драйвером.*

### 3. Добавление пути к драйверу в системную переменную PATH

- Открыть окно **Переменные среды**:  
  Перейти в **Панель управления** → **Система** → **О системе**.
- Выбрать **Дополнительные параметры системы**.
- Нажать кнопку **Переменные среды**.
- В списке системных переменных выбрать переменную **`Path`**.
- Нажать кнопку **Изменить**.
- Добавить новую запись: `C:\webdrivers`.
- Подтвердить изменения нажатием **ОК** во всех открытых окнах.

<br>![screenshots/webdrivers-path-variable.png](screenshots/webdrivers-path-variable.png)  
<br>*Скриншот 34: «Добавление пути C:\webdrivers в переменную окружения PATH» — окно редактирования переменной PATH с новой записью.*

### 5. Проверка работоспособности ChromeDriver

- Запустить новое окно командной строки (`cmd.exe`).
- Выполнить команду:
```bash
chromedriver --version
```
<br>![screenshots/chromedriver-version-check.png](screenshots/chromedriver-version-check.png)  
<br>*Скриншот 35: «Проверка установки ChromeDriver через командную строку» — вывод в командной строке с информацией о версии ChromeDriver.*

### 19. Начальная настройка глобальных параметров Git, редактора по умолчанию (опционально)

```bash

```

**Команда для настройки:**
```bash
git config --global core.editor notepad
```

### 20. Генерация SSH-ключа с установкой парольной фразы

Для безопасного подключения к удалённым репозиториям по протоколу SSH необходимо сгенерировать защищённую пару ключей.

Выполнены следующие команды в Git Bash:

#### Шаг 1: Переход в директорию SSH

```bash
cd ~/.ssh
```
Переход в домашнюю директорию пользователя для управления SSH-ключами.

#### Шаг 2: Просмотр существующих файлов

```bash
ls
```
Проверка существующих SSH-файлов: обнаружены директория **agent/** и файлы **known_hosts**.

#### Шаг 3: Генерация защищённого SSH-ключа

```bash
ssh-keygen -t ed25519 -C "tsybizova.anya@mail.ru"
```
Создание защищённой пары SSH-ключей с использованием современного алгоритма Ed25519

### 21. Настройка SSH-агента для управления ключами

Инициализация SSH-агента в текущей сессии терминала. Агент запущен с идентификатором процесса (PID) 1170.

#### Шаг 2: Добавление SSH-ключа в агент

```bash
ssh-add ~/.ssh/id_ed25519
```
Добавление сгенерированного ранее защищённого SSH-ключа в менеджер ключей.
